一句话原理
	•	Claude Code 作为“主代理”：它在终端里跑、自动把 CLAUDE.md 当成系统指令读进上下文，所以你能把“遇到大任务就把审阅/实现交给 Codex（通过 MCP）”这类规则固化进去，Claude 会稳定照做。 
	•	MCP 把 Codex“接成一个工具”：把 Codex 以 MCP server 的形式挂进 Claude Code 后，Claude 就能像用别的工具一样调用 Codex 来评审/实现/跑命令，而不是只用自家模型干到底。 
	•	免打断＝开启自动批准/跳过授权：Claude Code 这边用自动批准/危险跳过权限模式，Codex 那边用全自动/从不询问的审批策略（或等价 flag）。于是两个代理都不会停在“要不要我运行/改文件”的确认框，能在你离开的一小时里自循环把 TODO 做完。

复制即用：把 Codex 接到 Claude Code

# 1) 装好两边
curl -fsSL https://claude.ai/install.sh | bash            # 安装 Claude Code
npm i -g @openai/codex                                     # 安装 Codex CLI

（官方入口与定位：Claude Code 终端代理、Codex 终端代理。 ）

# 2) 把「Codex 当 MCP 服务器」接入 Claude Code
#   用 stdio 方式最直接：Claude 启 Codex 的 MCP 进程并连上
claude mcp add --transport stdio codex -- codex mcp-server

（Claude Code 支持通过 claude mcp add 增加 stdio 服务器；Codex 支持 codex mcp-server 以 MCP server 形式对外。 ）

# 3) 验证是否连上（在 Claude Code 会话里）
/mcp   # 能看到 codex 这个服务器与其工具

（/mcp 可查看活跃 MCP 连接。 )

在 CLAUDE.md 里“制度化”委托给 Codex

在仓库根目录放 CLAUDE.md，写清委托规则（Claude 会自动加载并严格遵循）：

# Orchestration rules
- 当任务属于“代码审阅、大型重构或长链路实现”，优先调用 **Codex（MCP）**。

- 先 /plan 再执行；单次循环失败即重试并附带失败上下文；以“测试全绿/PR就绪”为停止条件。

说明：Codex 的 MCP 工具端点名为 codex / codex-reply，并支持在调用时指定 approval-policy（如 never）与 sandbox（如 workspace-write）等参数，便于做到“无人值守”。

Claude Code 端（减少打断）
